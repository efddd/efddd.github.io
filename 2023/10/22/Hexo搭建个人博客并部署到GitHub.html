2023-10-22 Hexo

4生成SSH添加到Github

检查是否输对
git config user.name
git config user.email

创建SSH

ssh即简单来说是秘钥，id_rsa为主机的私人秘钥，不能给别人看

id_rsa.pub为主机公共秘钥，可以随便给别人看，当链接Github自己的账户时，它会根据公钥匹配你的私钥，当相互匹配时，就能顺利通过git上传你的文件到github了

此时已生成.ssh的文件夹，在电脑中可找到该文件夹，或者git bash中输入cat ~/.ssh/id_rsa.pub

到github部署密钥

打开github，点击头像

点击settings

点击SSH and GPG keys

点击New SSH key

将输入cat ~/.ssh/id_rsa.pub后输出的内容复制到框中

点击Add SSH key后

在git bash输入ssh -T git@github.com后输入yes后如下图即部署成功

总结

1.以更接近当前时间的教程为基准，老教程相关某些配置容易失效，卡住就是说明存在着失效的问题，需要自己探索解决。

2.按照例子（模板）的命令，或者配置文件标注都应该要校对，避免是因为文字输入出错的小问题而绕一大圈，低级问题尽量减少。

3.缓存文件容易影响进程，也就是相关配套的操作都要做，不能少，就好像环境配置要确保安装成功能显示版本号。

4.每次部署都要hexo c/g/s/d，s可以检查自己的博客成效，再d上传。

hexo主题安装（二）

1hexo init目录解析

node_modules

用于存放第三方模块的目录（依赖包）。

public

用于生成静态网站的目录

hexo g生成静态网站时，Hexo会将生成的HTML、CSS、JavaScript等文件存放在public文件夹中。

scaffolds

包含了模版文件，用于创建新的文章、页面。

source

博客的源文件目录，包含了文章、页面、图片等资源。

themes

用于存放主题文件的目录。在初始化时，Hexo会默认生成一个名为landscape的主题。

_config.yml

Hexo的主要配置文件，包含了博客的各种设置选项，如主题、插件、URL等。

2使用官方提供的主题

1下载主题

到主题 |六角形 (hexo.io)选择感兴趣的主题

按作者提供的方式（git clone到themes下/直接下载）。

2启用主题

在 Hexo 根目录的_config.yml文件中，将theme值修改为minimalism。

创建主题配置文件

在 Hexo 根目录下创建_config.minimalism.yml文件，并将主题作者提供的配置文件的所有内容复制进去，如未有特殊说明，所有的主题配置均在此文件修改。

3更新主题

在 Hexo 根目录下执行如下命令：

npm install hexo-theme-minimalism@latest

4重新部署文件

依次输入hexo c、hexo g、hexo s（测试）、hexo d。
